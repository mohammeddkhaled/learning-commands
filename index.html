Git Commands

  • git init: Initializes a new Git repository.
    This command creates a new .git directory in your project folder.
  • git clone [url]: Clones an existing repository from a URL.
    This command copies a repository from a remote source to your local machine.
  • git add [file]: Stages changes for the next commit.
    This command adds changes in the working directory to the staging area.
  • git commit -m "message": Commits staged changes with a descriptive message.
    This command records changes to the repository with a message.
  • git status: Displays the state of the working directory and staging area.
    This command shows which files are staged, unstaged, or untracked.
  • git log: Shows the commit history.
    This command displays a list of commits made in the repository.
  • git diff: Shows differences between commits or the working tree.
    This command compares changes between commits, branches, or the working directory.
  • git branch: Lists, creates, or deletes branches.
    This command manages branches in the repository.
  • git checkout [branch]: Switches to the specified branch.
    This command changes the working branch.
  • git merge [branch]: Merges the specified branch into the current branch.
    This command integrates changes from another branch.
  • git pull: Fetches from and integrates with another repository or branch.
    This command updates the local branch with changes from the remote repository.
  • git push: Uploads local branch commits to the remote repository.
    This command sends local changes to the remote repository.
  • git remote -v: Displays the remote repository URLs.
    This command shows the URLs of remote repositories.
  • git config --global user.name "Name": Sets the global username.
    This command configures the username for all repositories.
  • git config --global user.email "email@example.com": Sets the global email address.
    This command configures the email address for all repositories.
  • git reset [file]: Unstages a file while retaining changes.
    This command removes a file from the staging area.
  • git rm [file]: Removes a file from the working directory and stages it.
    This command deletes a file and stages the deletion.
  • git stash: Temporarily shelves changes for a clean working directory.
    This command saves changes for later use.
  • git tag [name]: Creates a tag reference.
    This command marks a specific point in the repository's history.
  • git fetch: Downloads objects and refs from another repository.
    This command retrieves updates from a remote repository without merging them.
  • git rebase [branch]: Reapplies commits on top of another base tip.
    This command integrates changes by moving commits to a new base.
  • git cherry-pick [commit]: Applies changes from a commit to the current branch.
    This command applies specific commits from one branch to another.
  • git revert [commit]: Reverts a commit by creating a new one.
    This command undoes changes by creating a new commit.
  • git clean -f: Removes untracked files from the working directory.
    This command deletes untracked files to clean the working directory.

GitLens Commands

  • GitLens: Toggle Line Blame: Shows/hides Git blame annotations for the current line.
    This command toggles Git blame annotations for the current line.
  • GitLens: Show Commit Details: Shows information about a specific commit.
    This command displays detailed information about a specific commit.
  • GitLens: Open Changes with Previous Revision: Compares file with its previous revision.
    This command compares the current file with its previous revision.
  • GitLens: Show File History: Displays the history of the current file.
    This command shows the commit history of the current file.
  • GitLens: Show Line History: Shows the history of a selected line.
    This command displays the commit history of a specific line in the file.
  • GitLens: Show Repository Status: Displays the status of your repository.
    This command shows the current status of the repository.
  • GitLens: Show Repositories View: Opens the GitLens repositories sidebar view.
    This command opens the repositories view in the GitLens sidebar.
  • GitLens: Show Commit Graph: Shows a visual representation of the commit history.
    This command displays a graphical representation of the commit history.
  • GitLens: Search Commits: Allows searching through commit messages and authors.
    This command enables searching for specific commits by message or author.
  • GitLens: Show Contributors: Lists contributors to the file or repository.
    This command lists all contributors to the current file or repository.

PowerShell Commands

  • Get-Help: Displays help about PowerShell commands.
    This command provides detailed help for PowerShell commands.
  • Get-Command: Lists all available commands.
    This command lists all available PowerShell commands.
  • Get-Process: Lists currently running processes.
    This command displays a list of all running processes on the system.
  • Stop-Process: Stops one or more running processes.
    This command terminates specified running processes.
  • Get-Service: Retrieves the status of services.
    This command displays the status of all services on the system.
  • Start-Service: Starts a service.
    This command starts a specified service.
  • Stop-Service: Stops a service.
    This command stops a specified service.
  • Set-ExecutionPolicy: Changes script execution policy.
    This command modifies the script execution policy for PowerShell.
  • Get-Content: Reads content from a file.
    This command retrieves the content of a specified file.
  • Set-Content: Writes or replaces content in a file.
    This command writes or replaces content in a specified file.
  • Add-Content: Appends content to a file.
    This command appends content to a specified file.
  • Copy-Item: Copies files or directories.
    This command copies files or directories to a new location.
  • Move-Item: Moves files or directories.
    This command moves files or directories to a new location.
  • Remove-Item: Deletes files or folders.
    This command deletes specified files or folders.
  • Rename-Item: Renames files or folders.
    This command renames specified files or folders.
  • New-Item: Creates a new file or directory.
    This command creates a new file or directory at the specified location.
  • Get-ChildItem: Lists files and folders in a directory.
    This command lists all files and folders in a specified directory.
  • Set-Location: Changes the current directory.
    This command changes the current working directory.
  • Get-Location: Displays the current directory.
    This command displays the current working directory.
  • Clear-Host: Clears the terminal screen.
    This command clears all content from the terminal screen.
  • Get-Alias: Lists aliases for cmdlets.
    This command lists all aliases for PowerShell cmdlets.
  • Export-Csv: Saves data to a CSV file.
    This command exports data to a specified CSV file.
  • Import-Csv: Reads data from a CSV file.
    This command imports data from a specified CSV file.
  • ConvertTo-Json: Converts objects to JSON format.
    This command converts objects into JSON format.
  • ConvertFrom-Json: Converts JSON data to objects.
    This command converts JSON data into PowerShell objects.
  • Test-Connection: Sends ICMP echo requests (ping).
    This command sends ICMP echo requests to test network connectivity.
  • Invoke-WebRequest: Sends HTTP/HTTPS requests.
    This command sends HTTP or HTTPS requests to a specified URL.
  • Start-Job: Runs a background job.
    This command starts a background job in PowerShell.
  • Get-Job: Gets background job information.
    This command retrieves information about running background jobs.
  • Receive-Job: Gets job results.
    This command retrieves the results of a completed background job.
  • Stop-Job: Stops a background job.
    This command stops a running background job.
